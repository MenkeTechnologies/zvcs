✓ Fixed — git add . at repo root (was: staged nothing)

In the real (large) zvcs repo, zvcs add . from root matched zero files while stock git staged them. Root cause: a bare . normalizes to a nil pathspec whose path is the literal "." (gix-pathspec/src/pattern.rs:112); gitoxide's dirwalk_iter then prefix-matches "." and emits only dot-prefixed entries (4 items) instead of the full tree (3188). Fix: at the repo root, rewrite a bare ././ pathspec to :/ (a . from a subdirectory normalizes to its prefix path and already walked correctly, so it is left untouched). Verified byte-parity with git — add -n . now stages 17 = git's 17; subdir/small-repo/real-stage cases unaffected. Commit 51cc283176.

Notes on classification

  • Recently upgraded (now real, beyond what a stale binary shows): push (send-pack.c port), reset --merge/--keep, merge diverged three-way + octopus, client-side hooks (pre-commit/commit-msg/pre-push/pre-rebase), rebase genuine-pick replay + --continue/--skip/--abort/--quit.
  • rerere is intentionally PARTIAL: the record/replay half is not faked because its conflict-id is a SHA-1 of git's exact xdl_merge output written into the git-shared rr-cache — a wrong id would silently corrupt a cache real git also reads.
  • -h is broken on ~24 commands (emit error: unknown switch 'h' or unsupported flag -h instead of git's usage-on-stdout at exit 129). Cosmetic but user-visible: add, am, apply, blame, branch, checkout, clone, commit, commit-tree, config, describe, diff, init, init-db, log, merge, pull, push, reflog, reset, restore, rev-parse, show, show-ref, stage, stash, status, subtree, symbolic-ref, whatchanged, upload-archive.
  • Helpers / not git subcommands: color (internal status-color module), push-proto (internal send-pack engine), upload-archive--writer / web--browse (internal helpers). jump, last-modified are zvcs extras.

Method: 8 parallel auditors read every pub fn body and ran git <cmd> -h for the git baseline; classifications are strict (real code-path bail ⇒ STUB regardless of flag literals or usage strings).
